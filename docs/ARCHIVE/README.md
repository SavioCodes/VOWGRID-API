# Documentation Archive

Files in this directory are preserved for traceability, handoff history, and audit context.

Use archived docs only when you need to understand how the repository evolved. Do not treat them as the active source of truth.

Current truth lives in:

- [Root README](../../README.md)
- [Documentation guide](../README.md)
- [Implementation status](../IMPLEMENTATION_STATUS.md)
- [Run guide](../RUN_GUIDE.md)
- [Architecture](../ARCHITECTURE.md)
- [API reference](../API_REFERENCE.md)

Archive groups:

- `docs/ARCHIVE/reports/` for historical integration and audit reports
- `docs/ARCHIVE/backend/` for older backend-pass docs that were superseded by canonical root docs
- `docs/ARCHIVE/handoffs/` for one-off handoff documents
- `docs/ARCHIVE/design/` for design status snapshots that no longer represent current product truth

Some archived files still contain internal links and assumptions from their original context. Treat those references as historical, not as current navigation.
