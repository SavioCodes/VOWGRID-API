# Auth Setup

## Scope

VowGrid now has real dashboard auth for human operators while keeping API keys for machine clients.

Implemented:

- Signup
- Login
- Logout
- Current session (`/v1/auth/me`)
- Protected `/app` routes in the web app
- Session-backed dashboard data fetching

Not implemented:

- Password reset
- Email verification
- Invites
- SSO
- Multi-workspace membership
- User-facing API key management

## Backend Model

Dashboard auth uses:

- `User.passwordHash`
- `User.lastLoginAt`
- `UserSession`

Session tokens are:

- generated by the API
- hashed before persistence
- sent back to the web app on signup/login
- stored by the web app in an HttpOnly cookie

## Environment

Required in `apps/api/.env`:

- `DATABASE_URL`
- `REDIS_URL`
- `SESSION_SECRET`
- `API_KEY_SALT`

Compatibility note:

- `JWT_SECRET` is still accepted as a fallback alias so older local env files do not break immediately
- the active design no longer uses JWT for dashboard auth

Required in `apps/web/.env.local`:

- `VOWGRID_API_BASE_URL=http://localhost:4000`

Optional in `apps/web/.env.local`:

- `VOWGRID_ENABLE_PROVISIONAL_DATA=true` to enable `/preview`

## Routes

Public auth routes:

- `POST /v1/auth/signup`
- `POST /v1/auth/login`

Session routes:

- `GET /v1/auth/me`
- `POST /v1/auth/logout`

Protected product routes:

- `/app/*`

## Web Flow

1. User submits the login or signup form in Next.js.
2. A server action calls the API auth endpoint.
3. The web app stores the returned session token in `vowgrid_dashboard_session`.
4. Protected server components read that cookie and call the API with `Authorization: Bearer <token>`.
5. If the session is missing or invalid, `/app` redirects to `/login`.

## Local Verification

Seeded local login:

- Email: `reviewer@vowgrid.local`
- Password: `vowgrid_local_password`

Checks:

- Open `http://localhost:3000/login`
- Log in with the seeded account
- Confirm `http://localhost:3000/app` renders the control plane
- Open `http://localhost:3000/login` again and confirm it redirects back to `/app`
- Log out and confirm `http://localhost:3000/app` redirects to `/login`

## Security Notes

- Passwords are hashed before storage
- Session tokens are hashed before persistence
- The browser only receives the opaque session token through an HttpOnly cookie
- API keys remain separate from dashboard auth and are not exposed through the web UI yet
